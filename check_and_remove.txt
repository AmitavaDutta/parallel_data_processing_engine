Task 4 — Include a GPU Memory Constraint Scenario
What they want

A situation where the GPU cannot fit the entire correlation matrix or dataset into memory, forcing a redesign of the algorithm.

Why this matters

GPU memory is limited (often 8–24 GB), while CPU RAM can be 32–128 GB+.

Example scenario

If:

N = 50,000 time series
T = 10,000 time steps

The correlation matrix size is:

N × N = 50,000² = 2.5 billion elements

Memory:

2.5B × 8 bytes ≈ 20 GB

This exceeds most GPU memory.

What you implement

Instead of computing the whole matrix at once:

Strategy 1 (Already done)
Full matrix computation
Strategy 2 (New)
Block-wise / tiled computation

Example:

for i in range(0, N, block_size):
    for j in range(0, N, block_size):
        compute correlation block(i,j)

Only small blocks are sent to GPU.

What you show in report

Explain:

Strategy	GPU Memory	Speed
Full matrix	High	Fast
Block-wise	Low	Slightly slower
